V1 kimlik kazanmış, sistem oturmuş — ama hero'da "ne yapan biri" ilk 2 saniyede görsel olarak henüz teslim edilmiyor, tipografide tek plak (Inter) çalıyor, bazı kopyalar ise her tasarımcının yazabileceği bir dille konuşuyor. V2'de replace değil, rebalance gerekli.

Elimizde ne var

9 tam vaka çalışması, 3 dil (EN/DE/TR), Braun/Rams tabanlı token sistemi, sağlam motion katmanı (Phase 1+2+3), Vercel oto-deploy, GDPR-uyumlu analytics, custom domain, %64 optimize edilmiş görsel yükü. Teknik alt zemin V1'e kadar solid.

Neyi geliştiriyoruz

Hero anatomisi (portre + iş), tipografi (display serif tanıtımı), kopya sesi (sharp), work grid asimetrisi, case study'nin 13-aşama yükünün rebalance'ı, motion'ın "gösterişten anlama" geçişi. Roadmap'te tek-bölümlü 7 sürüm.

01 — Envanter

Audit · 20 teknik + yapısal madde

Kod tabanının şu anki durumu, isim isim, dosya dosya. V2 kararlarını bu masaya yaslanacak.

#AlanŞu anki durum
01Framework Static HTML/CSS/JS. Framework yok. Basit ve hızlı. Trade-off: component reuse yok, 13 vaka sayfası aynı yapıyı tekrarlar (i18n key'lerinde çakışma riski, HTML de duplike).
02Folder structure ./ kök (HTML), assets/{css,js,fonts,img}, work/, pdf/, docs/ (private, .vercelignore), scripts/. Cruft: bodies/ (bozuk build sistemi), pdf/deck-v2.html (100KB, stale).
03Component hierarchy CSS-driven komponentler (~15): .project-card, .fw-card, .media-slot, .callout, .tl-item, .site-header, .primary-nav, .lang-switch, .theme-toggle, .consent-banner, .featured-work, .hero-portrait, .case-pager, .principle-grid, .dim-mark.
04Routing File-based. index.html, work/{slug}.html, pdf/index.html. Vercel vercel.json'da: /pdf → /pdf/index.html rewrite. /about, /about.html → /#about 301. Case-study prev/next static .case-pager ile.
05Styling system CSS custom properties (tokens.css, ~8 KB) + style.css (11 mantıksal bölüm başlıklı, ~32 KB). Inline styles az miktarda leak (index.html'de grid-column attr). Cascade layer yok, ama gerekmiyor.
06Animation libraries Kütüphane yok. Vanilla JS + CSS. IntersectionObserver reveal, rAF-throttled hero parallax, setInterval featured-work carousel, CSS transitions hover/reveal. prefers-reduced-motion her yerde honor. Bundle size sıfır.
07Typography Inter Variable (338 KB, self-hosted, tüm ağırlıklar) + JetBrains Mono Regular/Medium (~92 KB her biri). Hierarchy --fs-100 (11 px) → --fs-900 (72 px). Tek sans + tek mono — display serif yok. Editorial karakter zayıflığının kaynağı.
08Color tokens Braun grayscale (--paper, --ink-{900,700,500,300}, --line-{100,300}) + signal (#C1401F light, #E5654A dark). Dark tema flip'te --nihai-bg sabit koyu kalır (finale showcase). Tertiary/semantic renk yok — Braun disiplini burada bilinçli.
09Spacing system 8 px base rhythm. --sp-1 = 4 px → --sp-9 = 128 px. Consistent, ihlal yok. Layout gap'ları hep token ile.
10Hero implementation 2 kolon grid (text solda, portre sağda). Portre alpha PNG + --surface bg (dark modda krem — post PR #12). H1 max 4 satır, text-wrap: balance. Meta strip 3 kolon (Focus/Highlight/Background). Portrait parallax %14. Öne çıkan iş görseli yok.
11Project grid 3 katmanlı: featured-work (4 kart, cross-fade carousel), work grid (5 flagship .project-card), Selected Works (4 kısa vaka). Grid'de kartlar aynı boyut, hierarchy zayıf.
12Project detail 13 aşama template (Problem → Araştırma → Kriter → Konsept → Sketch → CAD → Mühendislik → DFM/DFA → Prototip → Test → İterasyon → Nihai → Kazanım). Sticky .case-subnav, .media-slot/.media-grid (>860 px'de breakout genişler), #nihai full-bleed Ken Burns. .case-pager prev/next.
13About section Single-page #about. Kicker + lede + process-oku (mono). .principle-grid 4 kart. Kart başlıklarında ses biraz jenerik ("Constraints stay visible" vs).
14Experience Timeline (5 .tl-item). TUS current, TUS intern, ITÜ ROV, Robotistan, ITÜ BSc. Sol rail (2 px line-300), tarih signal-red, reveal animasyonu. Yerinde.
15Contact Footer #contact. Formsubmit.co ile POST. Kicker + copy + email + LinkedIn + Behance + portfolyo linki. Toast ?sent=1 handler. Consent banner buraya karışmıyor.
16Responsive Breakpoints: 480, 640, 720, 860 px. Mobile nav full-screen overlay (backdrop-filter kaldırıldı, artık düzgün). Hero grid 1-col ≤720 px. Case media breakout ≤860 px kapanıyor.
17Build system Yok. Static dosyalar direkt serve. scripts/optimize_images.py (Python, manuel çağrı). CSS/JS minification yok. Hash-based cache-busting yok (yalnızca ?v= query, manuel bump).
18Deployment Vercel bedirhanckr/websiterepo main branch'ine bağlı. Push → auto-deploy. Custom domain bedirhancakiroglu.com. .vercelignore: docs/, scripts/, bodies/, *.md. vercel.json: redirects + cache headers (CSS/JS revalidate, images 1h, fonts 1yr).
19Performance risks Retroscent hero 3.2 MB PNG (grotesk). Portre 774 KB PNG hero'da eager load. i18n.js 168 KB her sayfaya. Featured-work 8 görsel yükler (4 primary + 4 alt). Hero image <picture> tag yok (WebP muadili var ama unused). GA gtag her sayfaya.
20Accessibility Skip-link ✓, focus-visible outline ✓, alt attr ✓, aria-label buton/nav ✓, semantic tags ✓, prefers-reduced-motion ✓. Zayıflıklar: renk kontrast --muted/--paper WCAG AA sınırında; bazı SVG'lerin aria-hidden'ı var, decorative label yok; PDF deck skip-to-content yok.

03 — V2 pusulası

Design Direction · site nasıl hissetmeli

Layer Design, Marc Newson ve Karim Rashid'in prensiplerinden öğrendiklerimizin özgün bir dile dönüştürülmüş hâli. Layout / renk / brand kopyalanmayacak.

Calm

Negatif alan yalvarır. Aynı ekranda 3 vurgu değil, 1 vurgu + 2 destek. Sayfa scroll'lanırken göze dinlenme molası verir.

Premium

Kısıtlı accent, ornament yok. Editorial yoğunluk — her satır bir şey söylüyor. Küçük detayların özenle çözülmüş olması hissedilir.

Engineered

Görünür oranlar, hizalamalar, teknik çizim motifleri. Dim-mark subtle kalır, referans olarak — süs değil.

Editorial

Display serif tanıtılır (Fraunces veya Iowan Old Style ailesi). Cömert kolonlar, ölçeklenmiş tipografi hierarchy. Portfolyo bir dergi gibi taranır.

Minimal

Gradient yok, box-shadow yok — Braun disiplini korunur. Motion sadece bilgi taşırken var; süsleme değil.

Industrial

Ortografik görüntüler, ürün fotoğrafı disiplini, teknik çizim izleri. "Bu adam bir mühendis" ilk ekranda söylenir.

04 — Iterasyon planı

Roadmap · V2.1 → V2.7

Her sürüm SADECE tek bir bölümü modifiye eder. Her sürüm ayrı bir PR, ayrı bir geri-alınabilirlik. Sıra ROI'ye göre.

V2.1

Hero — anatomi + tipografi + görsel yön

Hero'yu rebalance: portre chip'e küçülür (~120–160 px sağa yaslı); ana görsel Pioneer MKII veya REX render'ı büyük sinema oranı; H1 kısalır ve display serif'e geçer (Fraunces / Iowan Old Style variable); meta strip 1 satırlık orbital metaya. İlk 2 saniyede "ne yapan biri" görsel olarak teslim.

Süre · 1–1.5 gün Risk · Orta (kimliği sarsmadan denge) PR · tek
V2.2

Typography — display serif sistemi

V2.1'de hero'da tanıttığımız display face'i tüm h1/h2 hiyerarşisine yayarız. Body Inter kalır (okunabilirlik). Font-pair'in son teknik ayarları (letter-spacing, line-height, weight scale). Fraunces variable ise +opsz eksenini de kullanırız (küçük başlıklarda otomatik optical size). Bundle: +80–120 KB tek variable font.

Süre · 0.5 gün Risk · Düşük PR · tek
V2.3

Work grid — asimetrik hierarchy

5 flagship'te göze eşit ağırlık verilmiyor — 1 dev flagship (hero-scale sol), 2 orta supporting (sağa dikey), 2 daha küçük altta. Selected Works ayrı görsel bloğa (list-view + mikro thumbnail). Kart üstüne mikro chip: "Award / Shipped / Prototype". Featured-work strip ile duplication çözülür (birleştir ya da kaldır).

Süre · 1 gün Risk · Orta PR · tek
V2.4

Case study — 13 aşamayı esnet

Template katı kalmasın — her vaka kendi ideal aşama sayısı (7–13). Boş sketch slot'ları gizle (placeholder gösterme). Media grid orientation-aware: portre görsel için object-fit: contain. Sub-nav'ı vakanın gerçek aşamalarına göre üret. #nihai Ken Burns korunur.

Süre · 1–1.5 gün Risk · Orta (9 vaka × düzenleme) PR · tek
V2.5

About / Principles + Capabilities — sesin keskinleşmesi

Principle kartlarındaki kopya Bedirhan'ın gerçek sesine döndürülür (jenerik → spesifik). Örn: "Prototipsiz karar vermem"; "Rendering'i konseptin ispatı sanmam". 1 kart öne çıkar (hero-scale), 3 destek. Capabilities section skill blokları vaka referanslarıyla ilişkilendirilir (HiCAD → TUS balkon; SolidWorks → Delta).

Süre · 0.5 gün Risk · Düşük — sadece kopya + hierarchy PR · tek
V2.6

Footer / Contact — signature moment

Vedalaşmayı editorial hale getir. Rams alıntısı zaten var, daha görünür — kendi el yazısı SVG signature. Contact form küçülür (yardımcı), asıl CTA büyük "email me" buton. LinkedIn / Behance / portfolio alt satırda mono chips.

Süre · 0.5 gün Risk · Düşük PR · tek
V2.7

Motion refinement — anlam katsın, gösterişten çıksın

Parallax'ı reconsider (kaldır ya da daha subtle). Nav'a scroll-position aktif bölüm indikatörü (IntersectionObserver). Hero title reveal on load (subtle stagger, 400 ms tek seferlik). Featured-work carousel timing rebalance (4.8 s → 6 s, daha "premium" tempo). Case-subnav item hover'da geometrik dim-mark micro-anim.

Süre · 0.5–1 gün Risk · Düşük PR · tek
05 — Kritik sorunlar

Problems

Bugün canlı sitede öne çıkan, hemen dokunulmamış olan bulgular.

  • Hero'da iş görseli yok — ilk 2 saniyede "endüstriyel tasarımcı" görsel olarak teslim edilmiyor. En yüksek etkili tek sorun.
  • Tipografi tek plak — sadece Inter. Editorial voice zayıflığının kaynağı. Display serif'in yokluğu.
  • Kopya sesi bazı yerlerde jenerik — özellikle principles: "Constraints stay visible", "Designed as a system". Bedirhan'ın kendi sesi değil.
  • Retroscent hero 3.2 MB PNG — performance ciddi risk. 4G bağlantıda 5+ saniye. Optimize edilmeli.
  • Portre dark modda krem kart — çalışıyor ama engineered workaround hissi. Kimlik seçimi değil, uzlaşma. Renkli portre ile temeli değiştir.
  • Featured-work duplication — 4 flagship kart hem featured-work'te hem work grid'de. Ziyaretçi aynı 4 projeyi ard arda görür.
  • Case study 13 aşama katı — bazı vakalarda 8 aşama yeter (Retroscent zaten kısa, dürüst), bazıları placeholder ile geçiyor. Şablon esnetilmeli.
  • Kart hierarchy yok — hem work grid hem featured-work eşit ağırlıklı kartlarla dolu. En güçlü proje (Pioneer, TEKNOFEST) diğerleriyle aynı görsel-ağırlıkta.
  • Contact/footer vedalaşma zayıf — kompozisyon kesme noktası (site kapanışı) hissedilmiyor. Signature moment yok.
06 — Kapıda bekleyen kazanımlar

Opportunities

Elimizde hazır olan güçlü zeminler ve bunları nasıl unlock ederiz.

  • Kimlik zaten kazanılmış — Braun/Rams tabanlı token sistemi V1'in en güçlü sermayesi. V2 diline yayarken bu koruma altında kalır.
  • Doğrulanabilir sonuçlar — TEKNOFEST ödülü + İTÜ 2. + Robotistan'da satılan REX + FURNITUR 2023 (Retroscent). Bu 4 rozet henüz görsel olarak yeterince öne çıkmıyor.
  • Tam EN/DE/TR desteği — Almanya iş pazarı için nadir bir sinyal. Recruiter'e "Almanca konuşabilen tasarımcı" bilgisi mühimdir; hero'da tek satırlık bir referans olabilir.
  • Motion altyapısı sağlam — IntersectionObserver + reveal + carousel + parallax + Ken Burns. V2.7'de refinement için altyapı zaten var, sıfırdan başlanmayacak.
  • Case study derinliği bir moat — 9 tam vaka, 13 aşama şablonu, gerçek görseller. Rakiplerin çoğu 3–4 vakaya yer verirken Bedirhan'ın derinliği kredibilite.
  • Vercel + auto-deploy + Cloudflare-ready — teknik altzemin V2 boyunca engelsiz push edeceğimiz iterasyonlar için hazır.
  • PDF deck standalone silahı — recruiter'e "site + deck" birlikte gönderilebilir. Silahın gücü placeholder'lar dolunca artar.
07 — Uzak durulacaklar

Risks

V2'de ne yapmazsak kaybederiz.

  • Yıkıcı redesign kimliği bozar — Bedirhan'ın V1'de kazandığı Braun/Rams disiplinini heba etme riski. Her sürüm tek bölümlü olmalı; toplu revizyon yasak.
  • Tipografi swap Inter-detay sayfalarını sarsabilir — case study'lerin okunabilirliği inter'e alışkın. Display serif'i sadece h-display + h1/h2'de kullan; body ve UI Inter kalsın.
